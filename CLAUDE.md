## Project Overview

Otakuverse is a Flutter mobile social network for anime/manga fans. The app is in `otakuverse/` subdirectory. All Flutter commands must be run from within that directory.

## Commands

All commands run from `otakuverse/`:

```bash
# Install dependencies
flutter pub get

# Run app
flutter run

# Run all tests
flutter test

# Run a single test file
flutter test test/models/post_model_test.dart

# Static analysis
flutter analyze

# Generate mocks (mockito)
dart run build_runner build

# Build
flutter build apk       # Android
flutter build ios       # iOS
flutter build web       # Web

# Local Supabase backend
supabase start          # Starts local DB (port 54322), API (54321), Studio (54323)
supabase db reset       # Reset DB and apply seed.sql
supabase stop

# Generate launcher icons / splash
dart run flutter_launcher_icons
dart run flutter_native_splash:create
```

## Architecture

### Layer Structure

```
lib/
├── main.dart              # App entry: Firebase + Supabase init, GetX routes, global services
├── core/                  # App-wide infrastructure
│   ├── constants/         # AppColors, AppTextStyles, Dimensions, Assets paths
│   ├── services/          # ConnectivityService, RealtimeService, PushNotificationService, AudioManager
│   ├── theme/             # AppTheme.dark (Material 3, dark-only)
│   ├── utils/             # extensions.dart, validators.dart, helpers.dart, DateFormatter
│   └── widgets/           # AppBarWidget, AppBottomNav (reused across features)
├── features/              # Feature modules — each with bindings/controllers/models/repositories/services/screens/widgets
│   ├── auth/              # Sign in/up, onboarding (genre selection), Google Sign-In
│   ├── feed/              # Posts, comments, likes, bookmarks — the main feed
│   ├── profile/           # User profiles, follow system, Otaku rank/level
│   ├── message/           # Direct messaging with real-time updates
│   ├── notification/      # Push notifications + in-app activity feed
│   ├── search/            # User and content search
│   ├── stories/           # Instagram-style stories
│   ├── explore/           # Discovery/explore feed
│   ├── community/         # Placeholder (coming soon)
│   └── navigation/        # NavigationPage — IndexedStack with 5 bottom tabs
└── shared/
    ├── config/            # ApiConfig: Supabase URL/key, REST endpoint builders, isProduction flag
    ├── models/            # UserModel (shared across features)
    └── services/          # StorageService (SharedPreferences + SecureStorage), StorageUploadService
```

### State Management: GetX

- **Bindings** inject controllers lazily when a route is entered. `FeedBinding` injects `PostsController`, `FollowController`, `CommentController`, `NotificationController`, `BookmarkController`.
- **Global services** (`RealtimeService`, `StoryController`, `ConnectivityService`) are registered as permanent in `main.dart`.
- Access pattern: `Get.find<SomeController>()` or `SomeService.to` (static getter on `GetxService`).
- Routes are defined in `main.dart` → `Routes` constants + `_pages` list on `OtakuverseApp`.

### Backend: Supabase

- Supabase client is used directly — no ORM. Models parse with `.fromJson()` factories.
- **Real-time**: `RealtimeService` subscribes to Postgres changes on `posts`, `likes`, `comments`, `follows`, `bookmarks` tables and triggers controller refreshes.
- **Auth flow**: Sign up creates a row in `profiles` via a Supabase database trigger. Session is checked at startup to determine initial route (`/login` vs `/home`).
- **Storage**: Media uploads go through `StorageUploadService` to Supabase Storage buckets.
- **Edge Functions**: `send-notification` function in `supabase/functions/send-notification/`.

### Environment / Config

- `ApiConfig.isProduction = false` in [lib/shared/config/api_config.dart](otakuverse/lib/shared/config/api_config.dart) — flip to `true` for production builds.
- `baseUrl` auto-selects Android emulator (`10.0.2.2`), iOS simulator (`localhost`), or physical device IP based on `defaultTargetPlatform`. Update the physical device IP when needed.
- Supabase URL and anon key are hardcoded in `ApiConfig`. Firebase config is in `firebase_options.dart` (auto-generated by Firebase CLI).

### Theming & Typography

- Dark-only app — `AppTheme.dark` is applied for both `theme` and `darkTheme`.
- Color palette in `AppColors` (`bgPrimary = #0D0D14`, accent red `#E01A3C`).
- Fonts: **Bangers** (titles/manga feel), **Nunito** (body text), **JetBrainsMono** (stats/ranks).
- Screen sizing via `flutter_screenutil` — always use `.w`, `.h`, `.sp` suffixes.

### Otaku Rank System

`ProfileModel` has a `rank`/`level`/`xp` system (Novice → Sensei → Kami). `levelProgress` returns a `0.0–1.0` float used for progress bars. XP thresholds are computed in the model.

## Testing

Tests live in `otakuverse/test/`:
- `models/` — unit tests for `PostModel`, `CommentModel`, `ProfileModel`
- `controllers/` — `AuthController`, `PostsController` (mockito mocks generated via `build_runner`)
- `security/security_test.dart` — input validation and security checks
- `widget_test.dart` — smoke tests for all core model JSON parsing
- `helpers/fixtures.dart` — shared mock JSON factories (`postJson()`, `profileJson()`, `commentJson()`)

When adding a new model test, use `fixtures.dart` helpers rather than inlining raw JSON.
